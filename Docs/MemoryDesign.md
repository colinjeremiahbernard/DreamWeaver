DreamWeaver stores every creative idea as a persistent evolving universe inside a structured data system built around five core entities: User, Project, Universe, AgentRun, and CreativeMemory. The system is designed so that nothing generated by the AI is ephemeral; every decision becomes part of a long-lived creative state that can be revisited, revised, and extended over time.

A User represents a person interacting with DreamWeaver and owns multiple Projects. A Project is the entry point for a creative idea and contains the original user input along with metadata such as title, status, and timestamps. Each Project can spawn one or more Universes, where a Universe represents a fully structured creative world generated from the initial idea. A Universe is not just a final output but a living container for story, world-building, characters, tone, and visual identity.

Every Universe maintains a CreativeMemory object, which is the most important persistent structure in the system. CreativeMemory stores the defining attributes of the universe such as theme, tone, audience, world rules, character relationships, narrative state, visual style, and constraints. This memory is continuously updated as AI agents contribute new information or refine existing elements. It acts as the single source of truth that all AI agents must respect.

AI activity inside DreamWeaver is tracked through AgentRun entries. Every time an AI agent executes, whether it is the Creative Director, Story Agent, World Agent, Character Agent, Visual Agent, or Critic Agent, a corresponding AgentRun record is created. This record stores the input context used, the output generated, the type of agent that ran, and the associated Universe and Project. This allows full traceability of how any piece of creative content was produced.

The relationship between these components is hierarchical but persistent. A User contains Projects. Each Project contains Universes. Each Universe contains CreativeMemory and many AgentRuns. CreativeMemory is not static; it evolves through multiple agent interactions. The Creative Director initializes it, production agents expand it, and the Critic Agent modifies it when inconsistencies or improvements are required.

As the system operates, AgentRuns are executed in real time. Outputs are streamed to the frontend, stored in the database, and then used to update CreativeMemory. This creates a feedback loop where each generation influences the next, ensuring continuity and coherence across the entire creative universe.

The key principle of this system is that nothing is lost. Every creative decision, whether accepted, revised, or rejected, is stored and traceable. This enables users to revisit earlier versions of their universe, branch creative directions, or regenerate specific components without losing consistency.

Unlike traditional AI systems that generate isolated responses, DreamWeaver treats each creative output as part of a persistent evolving structure. This transforms the system from a stateless generator into a stateful creative engine capable of long-term narrative and world-building continuity.
